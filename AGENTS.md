# AGENTS

## 기본 원칙

- 기존 기능 및 디자인 유지
- 최소 수정 원칙 (Minimal Change)
- 관련 없는 파일 수정 금지
- 전체 파일 재작성 금지

## 코드 작성

- 중복 코드X  리팩토링 중요
- 주석 X

## 구조

- 현재 수정중인 ts, tsx 파일은 500줄 이상이면 디렉토리 분리

## DB

- SQL 변경은 sql.txt 맨 아래 추가(Supabase cli 사용안함)
- Supabase SQL 작성 시 (RLS, Policy : 필요시), Grant 포함
- 불필요한 SELECT X , 필요한 컬럼만 조회 가능하면 SELECT * 사용 지양
- 검색이 많은 컬럼은 Index를 고려
- 여러 SQL이 하나의 작업이라면 Transaction을 사용
- 필요한 경우 JOIN으로 한 번에 조회하는 것을 우선 고려
- 같은 데이터를 반복 조회 X
- FK, UNIQUE 등 데이터 무결성을 고려

## Cache

- 조회가 잦은 데이터는 필요한 경우 캐시를 사용

## 작업

- 필요한 파일만 읽고 필요한 부분만 참고
- 사용자가 요청하지 않은 변경 금지

## 출력 원칙

- 전체 파일을 다시 출력 X
- 변경된 코드만 출력
- 불필요한 설명 X 필요 시 1~2줄만 설명
- 수정한 파일의 경로 표시
- 한 줄 요약(필요 시)
