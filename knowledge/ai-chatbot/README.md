# AI 챗봇 문서 가이드

이 폴더의 `.md` 파일은 DOOGO CONNECT AI 상담 봇이 답변할 때 참고하는 지식 문서입니다.

## 기본 원칙

- 문서 1개에는 주제 1개만 담습니다.
- 제목은 첫 줄에 `# 제목` 형식으로 작성합니다.
- 문서 내용은 실제 답변 근거가 되는 정보만 넣습니다.
- 불확실한 내용, 임시 메모, 작업용 문장은 넣지 않습니다.

## 선택 메타데이터

필요하면 문서 상단에 아래 2가지를 넣을 수 있습니다.

```md
KEYWORDS: 견적, MOQ, 포인트, estimate
PATHS: /estimate, /my-connect
```

- `KEYWORDS:`  
  챗봇이 질문과 문서를 매칭할 때 참고하는 키워드입니다. 쉼표로 구분합니다.

- `PATHS:`  
  특정 페이지 경로에서 더 잘 선택되게 만들고 싶을 때 사용합니다. 쉼표로 구분합니다.

## 권장 폴더 구조

- `doogo-connect-prompts/`
  두고커넥트 서비스, 견적, 제조, 입점, FAQ 등 플랫폼 공통 문서

- `doogobio-nz-prompts/`
  두고바이오 NZ 전용 문서

## 문서 작성 예시

```md
KEYWORDS: 제조사 입점, 입점 절차, 제휴
PATHS: /

# 제조사 입점 안내

제조사 입점 신청이 가능합니다.

입점 절차:
STEP 01. 제조사 입점 문의 신청
STEP 02. 공장 실사 또는 검토
STEP 03. 계약 및 등록

문의:
contact@doogoconnect.com
```

## 작성 팁

- 질문이 `입점 문의`, `연락처`, `이메일`처럼 들어올 수 있으면 해당 단어를 문서에 직접 넣습니다.
- 질문이 `방법`, `절차`, `어떻게`처럼 들어올 수 있으면 `절차`, `STEP`, `신청 방법` 표현도 같이 넣습니다.
- 너무 긴 문서는 여러 파일로 나누는 것이 좋습니다.
- 같은 내용이 여러 문서에 중복되면 챗봇 선택이 흔들릴 수 있으니 가능한 한 한 문서에만 유지합니다.
